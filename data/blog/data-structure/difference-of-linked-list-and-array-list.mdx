# Linked List와 Array List 차이

## 메모리 할당

### ArrayList

연속된 메모리 공간에 데이터를 저장합니다. 크기를 동적으로 조절하기 위해 배열을 사용하며, 배열의 크기를 초과할 경우 더 큰 공간을 할당하고 이전 데이터를 복사해야 할 수 있습니다.

### Linked List

비연속적인 메모리 블록에 데이터를 저장합니다. 각 노드는 데이터와 다음 노드를 가리키는 포인터(링크)로 이루어져 있습니다.

## 접근 시간

### ArrayList
인덱스를 사용하여 빠르게 요소에 접근할 수 있습니다. 배열 기반 구조이기 때문에 인덱스를 이용한 랜덤 액세스가 상수 시간에 가능합니다.

### Linked List
특정 위치에 있는 요소에 접근하기 위해서는 처음부터 순차적으로 찾아가야 합니다. 따라서 특정 인덱스에 접근하는 데에는 O(n) 시간이 소요됩니다.

## 크기 변경

### ArrayList
배열 기반으로 크기를 동적으로 조절할 수 있습니다. 크기를 늘리거나 줄일 때마다 배열의 복사가 필요할 수 있습니다.

### Linked List
크기를 동적으로 조절하기 쉽습니다. 노드를 추가하거나 삭제할 때 링크를 조절하면 되므로 배열처럼 복사가 필요하지 않습니다.

## 삽입 및 삭제

### ArrayList
특정 인덱스에 요소를 삽입하거나 삭제할 때, 해당 인덱스 이후의 모든 요소를 이동시켜야 합니다. 이로 인해 삽입 및 삭제 연산이 느릴 수 있습니다.

### Linked List
특정 노드를 가리키는 링크를 조절함으로써 삽입 및 삭제를 상대적으로 빠르게 수행할 수 있습니다.

## 메모리 사용

### ArrayList
요소마다 추가적인 메모리가 필요합니다. 배열의 크기를 동적으로 조절해야 할 때 더 많은 메모리가 필요할 수 있습니다.

### Linked List
각 노드는 데이터와 다음 노드를 가리키는 포인터만을 저장하므로 메모리 사용이 좀 더 효율적일 수 있습니다.

`따라서 어떤 자료 구조를 선택할지는 사용하고자 하는 상황과 연산에 따라 다르며, 각각의 장단점을 고려하여 선택해야 합니다.`
